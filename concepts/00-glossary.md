# Glossary

The vocabulary this agent uses. When any of these terms shows up for the first time in a consultation, the agent explains it in one line before using it.

| Term | Meaning |
|---|---|
| **MECE** | Mutually Exclusive & Collectively Exhaustive. No overlaps, no gaps. **CE (no gaps) matters more than ME** — missing a case is worse than double-counting one. |
| **Structuring** | Turning a messy problem into order through *break down → group → sort*. The hard part is the breaking down. |
| **Logic Tree** | A problem decomposed into a MECE hierarchy — a tree of sub-problems. |
| **Top-down** | Deductive thinking: split a big vague problem, form a hypothesis, test it. The agent's default mode. |
| **Bottom-up** | Inductive thinking: combine small facts into a pattern, then an insight. |
| **Hypothesis** | Not a question — a **conclusion-shaped claim** ("X is true!") that can be proven or killed. |
| **CSAC** | Clarification → Structuring → Analysis → Conclusion. The four-stage problem-solving arc. |
| **Clarification** | The first stage: re-asking "is this even the real problem?" before solving anything. |
| **Align** | A shared agreement on the problem's scope and framing, reached before research starts. |
| **Initial research** | Shallow scouting — just enough to form a hypothesis. Not deep validation. |
| **Key Question (KQ)** | The question whose answer proves or kills a hypothesis. The best conclusion you can pre-write given what you know now. |
| **Sub Question (SQ)** | A KQ broken into a concretely researchable piece. |
| **Answer First (AF)** | Deciding *in advance* the shape of the answer and the data it would live in — and what insight it would yield — before you go looking. |
| **So what? / Why so?** | *So what?* pushes a fact up into a conclusion. *Why so?* drills a conclusion down to its grounds. |
| **Finding** | What emerges when validated KQs converge. Findings stack into higher-level insight. |
| **Pyramid Principle** | The write-up shape: one recommendation on top, MECE arguments below, evidence at the base. A reader descends only where unconvinced. Checked with *Why so?* going down, *So what?* going up. |
| **SCQA** | Situation → Complication → Question → Answer — the four-sentence opening that earns a conclusion its context before the pyramid defends it. |
| **KSF** | Key Success Factor — what an industry or company must get right to win. |
| **Framework** | A structural lens (3C, 4P, SWOT, …) for looking at a problem MECE-ly. A tool for *organizing*, not a checklist to fill. |
